# 更新日志

以后每次功能更新、行为变化、部署方式调整，都在这里记录。

## 2026-06-12

- 新增一周过期的搜索结果缓存，并在响应中返回缓存命中信息。
- 优化 Linux 上 Playwright/Chromium 的启动参数，增加登录和搜索脚本日志。
- 将通宝换算规则调整为 `1 人民币 = 100 通宝`。
- 搜索页新增等待进度提示，避免长时间搜索时看起来像页面卡住。
- 前端页面移除原始 JSON 展示，JSON 仍可通过 `/api/search/` 获取。
- 搜索页改为前端异步请求 `/api/search/`，提交搜索和打开带 `q` 的链接时都会先显示进度条。
- 为前端脚本增加版本参数，避免浏览器继续使用旧的静态文件缓存。
- 优化搜索速度：拦截图片、媒体、字体和广告资源，并发打开详情页抓取价格。
- 新增 HTTP 快速搜索路径，优先复用登录 Cookie 直接请求 Discuz 搜索和详情页，失败时自动回退 Playwright。
- 每条搜索结果从帖子详情页提取最多 3 张正文预览图，并在前端结果卡片中展示。
- 新增本更新日志，后续更新都同步记录。

## 2026-06-25

- 新增 Qinglanhua 搜索来源，基于公开首页和版块页做关键词匹配。
- 多来源结果改为轮询合并，避免单个来源占满前三条结果。

## 2026-06-25

- 增加生产环境静态文件配置 `STATIC_ROOT`，修复服务器执行 `collectstatic` 时报未配置静态文件目录的问题。
- 支持通过环境变量配置 `DJANGO_SECRET_KEY`、`DJANGO_DEBUG` 和 `DJANGO_ALLOWED_HOSTS`，方便服务器部署时覆盖默认配置。

## 2026-06-25

- 新增 `deploy_restart.sh`，用于服务器更新代码后一键拉取、安装依赖、迁移数据库、收集静态文件并重启 `search-console` 服务。

## 2026-06-25

- 将普通用户默认搜索结果数从 3 条调整为 10 条，VIP 默认上限调整为 20 条。
- 新增 `sync_search_sources` 管理命令，并在 `deploy_restart.sh` 中自动同步默认搜索源，避免服务器数据库仍只保留旧来源。
- 更新首页搜索说明，移除“当前单源 TK55TK”的旧提示。

## 2026-06-25

- 修复历史 VIP 配置可能把搜索结果继续限制为 3 条的问题，现在有效搜索上限不会低于普通默认上限。
- 将 TK55TK 和 Qinglanhua 命令行脚本默认搜索条数同步调整为 10 条，并提升搜索缓存版本使旧的 3 条缓存失效。

## 2026-06-25

- 优化前端搜索请求的错误处理：当 `/api/search/` 返回 HTML 错误页时显示 HTTP 状态和页面标题，避免直接出现 `Unexpected token '<'`。
- 更新前端脚本版本号，减少浏览器缓存旧脚本导致的问题。

## 2026-06-25

- 修复 TK55TK 登录态缺失时搜索请求触发 `SystemExit`，导致 Gunicorn 返回 HTML 500 错误页的问题；现在会作为普通搜索源错误处理并返回 JSON。

## 2026-06-25

- TK55TK 搜索在缺少 `tk55tk_output/storage_state.json` 时会自动使用配置文件或环境变量中的账号密码登录并生成登录态，然后继续搜索。
- `search_tk55tk.py` 新增 `--no-auto-login` 参数，用于需要手动管理登录态时关闭自动登录。

## 2026-06-25

- 搜索缓存版本提升到 7，清掉此前可能缓存的一周 Qinglanhua-only 半残结果。
- 当任一搜索源失败时不再缓存部分结果，并在前端展示每个来源的结果数量或失败状态。
- 多来源聚合顺序改为按默认来源配置排序，优先展示 TK55TK，再展示 Qinglanhua。

## 2026-06-25

- 新增访问日志 `AccessLog`，记录动态请求的 IP、路径、方法、状态码、User-Agent、Referer 和访问时间。
- 首页新增访问统计，展示累计访问次数和独立访客数量。
- 后台新增访问日志管理页，便于查看最近访问记录。
